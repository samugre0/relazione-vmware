Cos'è VMware: Architettura e Componenti

VMware rappresenta la soluzione di virtualizzazione enterprise più diffusa. Utilizzeremo VMware vSphere, una piattaforma di cloud computing che permette di consolidare l'infrastruttura fisica in una struttura virtuale flessibile.

Definizione di Hypervisor

Un hypervisor (o Virtual Machine Monitor - VMM) è un software che crea ed esegue macchine virtuali. Esistono due tipologie:

  • Type 1 (Bare-metal): Installato direttamente sull'hardware fisico (es. VMware ESXi). Offre prestazioni superiori.
  • Type 2 (Hosted): Installato sopra un sistema operativo convenzionale (es. VMware Workstation).

VMware ESXi è un hypervisor di Tipo 1, che sostituisce completamente il sistema operativo tradizionale sul server fisico.

Componenti fondamentali

Figura 1: Interfaccia VMware vSphere Client 7.0

1. Networking

Gestione di Virtual Switches (vSwitch), Port Groups con policy comuni (VLAN, sicurezza), VMkernel NIC per traffico di gestione (vMotion, FT), e Uplink verso switch fisici.

2. Storage

Datastores dove risiedono i file delle VM (formati VMFS, NFS), Virtual Disks (VMDK), e funzionalità come Storage vMotion per spostare VM senza downtime.

3. Virtual Machines

Centro di controllo per monitorare stato acceso/spento, allocazione risorse (CPU, RAM), indirizzi IP, e strumenti di snapshot per backup.

Nota: VMware vCenter Server è il sistema di gestione centralizzata che permette di amministrare multipli host ESXi, con funzionalità avanzate come vMotion (migrazione live), DRS (bilanciamento carico) e HA (failover automatico).

Vantaggi della virtualizzazione

  • Consolidamento server: Riduzione del numero di server fisici necessari
  • Isolamento: Ogni VM è indipendente, un crash non influenza le altre
  • Portabilità: Le VM sono file che possono essere spostati tra hardware diversi
  • Risparmio energetico: Minore consumo elettrico e raffreddamento
  • Rapidità deployment: Creazione di nuovi server in minuti anziché giorni

Procedura Dettagliata: Creazione VM

Fase 1: Configurazione Iniziale

Passo 1: Selezione compatibilità hypervisor (es. ESXi 7.0) per determinare funzionalità avanzate disponibili.
Passo 2: Selezione sistema operativo guest: "Microsoft Windows" → "Windows Server 2022".
Figura 2: New Virtual Machine Wizard - Scelta Typical vs Custom

Fase 2: Allocazione Risorse Hardware

CPU

Per Windows Server 2022 in ambiente lab: 1-2 socket virtuali, 2-4 core per socket (totale 2-4 vCPU). Non superare il numero di core fisici disponibili.

RAM

Windows Server 2022 Desktop Experience: minimo 4 GB, consigliati 6-8 GB. Versione Core: minimo 2 GB.

Storage

Dimensione: 60-100 GB. Provisioning:

  • Thick Provision: Allocazione immediata (migliore performance)
  • Thin Provision: Allocazione su richiesta (risparmio spazio)

Fase 3: Scelta Edizione Windows Server

Implementazione Active Directory

Teoria: Domain Controller

Un Domain Controller (DC) ospita il database Active Directory (NTDS.DIT) e fornisce servizi di autenticazione Kerberos/LDAP. Funzioni:

  • Autentica accessi utente (logon)
  • Gestisce politiche di sicurezza (Group Policy)
  • Mantiene catalogo globale (Global Catalog)
  • Gestisce risoluzione nomi tramite DNS integrato

Fase 1: Installazione Ruolo

Server Manager → Manage → Add Roles and Features:

  1. Server Selection: selezionare server locale
  2. Server Roles: spuntare Active Directory Domain Services
  3. Aggiungere features richieste (Group Policy Management)
  4. Completare wizard e attendere installazione
Figura 8: Server Manager Dashboard
Figura 9: Wizard - Selezione AD DS
A installazione completata, NON RIAVVIARE. Completare prima la promozione a DC.

Fase 2: Promozione a DC

Flag giallo in Server Manager → "Promote this server to a domain controller":

  • Deployment Configuration: Add a new forest
  • Root domain name: dc-4l
  • Forest functional level: Windows Server 2016
  • Password DSRM: [impostare password complessa]
  • DNS Options: Create DNS delegation (se richiesto)
  • Paths: default (C:\Windows\NTDS, C:\Windows\SYSVOL)
Figura 10: Promozione a Domain Controller

Cliccare Install. Il server si riavvierà automaticamente.

Configurazione DNS

Active Directory è strettamente integrato con DNS. I client usano DNS per trovare i Domain Controller (record SRV) e risolvere nomi host.

Regola d'oro: Il DC deve puntare a sé stesso (127.0.0.1) come DNS primario. Mai usare DNS esterni come primario su un DC!

Configurazione Forwarders

I forwarders permettono al DNS interno di risolvere nomi Internet interrogando server esterni:

1. DNS Manager → Tasto destro server → Properties
2. Tab Forwarders → Edit
3. Add: 8.8.8.8 (Google) e/o 1.1.1.1 (Cloudflare)
4. OK per applicare
Figura 12: DNS Manager - Configurazione Forwarders

Zone Create Automaticamente

  • Forward Lookup Zone: dc-4l (record A, NS, SOA, SRV)
  • Reverse Lookup Zone: 0.17.172.in-addr.arpa (risoluzione IP→nome)
  • _msdcs.dc-4l: Localizzazione servizi AD
Verifica: Eseguire nslookup dc-4l e nslookup -type=SRV _ldap._tcp.dc._msdcs.dc-4l per confermare che il DC sia raggiungibile.

Join dei Client al Dominio

Prerequisiti Client

  • Windows 10 Pro (la Home non supporta join a dominio)
  • Connettività di rete con DC (ping 172.17.0.88)
  • DNS configurato: primario 172.17.0.88
Figura 13: System Properties - Unione al dominio
Se il client usa DNS esterno (8.8.8.8), darà errore "DNS name does not exist" durante il join.

Procedura Join

  1. Settings → System → About → Advanced system settings
  2. Tab Computer Name → Change
  3. Selezionare Domain e inserire: dc-4l
  4. Autenticare con credenziali amministratore dominio
  5. Riavviare
Figura 14: Domain Join - Inserimento credenziali amministratore

Configurazione Condivisione "Area Tecnici"

Sul server:

  1. Creare cartella C:\Condivisioni\AreaTecnici
  2. Properties → Sharing → Advanced Sharing
  3. Permissions: rimuovere Everyone, aggiungere GR_Tecnici (Full Control)
  4. Security (NTFS): Disable inheritance → Convert → Aggiungere GR_Tecnici

Test Scenari

❌ Utente Standard (Mario Rossi) Accesso a \\dc-4l\AreaTecnici
Risultato: Accesso negato
Non membro di GR_Tecnici
✅ Utente Tecnico Accesso a \\dc-4l\AreaTecnici
Risultato: Accesso consentito
Membro di GR_Tecnici con permessi Full Control

Test UAC (User Account Control)

Tentare installazione software con utente tecnico: appare prompt UAC che richiede credenziali amministratore, confermando che i diritti di dominio non equivalgono a diritti amministrativi locali.

Errori Comuni e Soluzioni

Figura 15: Event Viewer - Visualizzazione errori AD e DNS
Problema Causa Soluzione
"Domain doesn't exist" DNS errato Impostare DNS 172.17.0.88
"Access denied" su share Permessi insufficienti Verificare membership gruppo

Figura 16: Command Prompt - Verifica connettività e risoluzione nomi

Termini usati nel lab

  • OU (Unità Organizzativa): Le "cartelle" in AD dove mettiamo utenti e gruppi per organizzarli meglio
  • Gruppo di Sicurezza: Un contenitore di utenti a cui assegniamo gli stessi permessi (tipo GR_Tecnici)
  • DNS: Il sistema che traduce i nomi (come dc-4l) in indirizzi IP
  • Share/Condivisione: Cartella di rete accessibile da altri PC (come Area Tecnici)
  • Join al dominio: L'operazione che fa entrare un PC Windows nel dominio dc-4l
